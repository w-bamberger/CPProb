Detailed Description

Random vector that represents a probability table.

A probability table usually shows in the left column all possible values of a discrete random variable; they are the outcomes of an experiment. The right column contains the probability assigned with the value of the discrete random variable. RandomProbabilities represents such a table, in which the probabilities in the right column are random variables. These random variables are constraint in the way that they must sum up to 1. So a certain object of this class is a certain realisation of the random variable containing specific probability values that sum up to 1.

A probability table can be well represented as a mapping from the value of the outcome to its probability. So this class implements a map interface as defined in the C++ Standard Template Library. In addition, it has a name assigned and contains methods for output streaming and random initialisation.

Generates a random value in the range of the random variable and assigns it to itself.

This may be useful for certain algorithms, e.g. to generate start values.

The argument is a C++ random number engine. You can find those engines in the standard TR1 header random. They are uniform pseudo-random number generators. From these engine, non-uniform number generators can be derived. Here the engine must be of the type defined in the project header random_number_engine.h. It is best to use the global variable vanet::random_number_engine. In the future, this argument will be removed.

std::pair<iterator, bool> vanet::RandomProbabilities::insert ( const value_type &  row) [inline]

Tries to insert a new row in this probability table.

It only inserts the row if its event (row->first) is not yet listed in the probability table. Then it returns an iterator to the new row and the boolean true. If the event row-first is already part of the probability table, false and an iterator to the already existing row for the given event is returned.

This method is here for compatibility with standard maps. Maybe it is easier to use set() instead.

Requires:
  • 0.0 <= row->second <= 1.0.
Ensures:
  • If row->first is not part of this probability table, insert(row) is equivalent to set(row->first, row->second).
  • insert(row)->first->first == row->first
  • If insert(row)->second == true, insert(row)->first == row
Parameters:
rowthe event-probability pair to insert
Returns:
an iterator to the event entry and and boolean that says whether a new entry has been created or not
Exceptions:
std::bad_allocthe memory for inserting a new element could not be requested.

virtual const std::string& vanet::RandomProbabilities::name ( ) const [inline, virtual]

Provides the name of the experiment this variable is assigned with.

The name of a random variable might be something like the FirstDice, SecondDice, Flavor, Wrapper etc. These names are mostly used for to generate understandable output.

In addition, the name of a random variable refers to a set of possible outcome values. The variable can only take on values of this set. For example, if one variable with the name GreenWrapper is a boolean variable (set of outcomes Ω = {true, false}), then another variable with the same name refers to the same set of outcomes.

Returns:
a descriptive name associated with this variable

Accesses the probability of the event var.

If var is not part of the probability table yet, it is inserted with probability 0. This method returns a reference to the probability for read and write access.

Ensures:
  • If var is part of this probability table, operator[](var), find(var)->second and at(var) provide all the same reference.
  • If var is not part of the probability table and operator[](var) is called, find(var)->second is valid and returns 0.0f.

Modifies the probability associated with the event var.

If var was not yet part of this probability table, it is inserted with the given probability.

This method is equivalent to operator[], except that it does not require default constructability but copy constructability. It is here, to be compatible with other distributions for which operator[] is not possible.

Requires:
  • 0.0 <= probability <= 1.0
Ensures:
  • set(var,probability) is equivalent to (*this)[var] = probability.
